# ROADMAP.md — מסמך המשימות של החזון

זהו **קובץ המשימות שכל סוכן AI עובד לפיו**. כל שלב מפורט בשתי רמות: **צד לקוח (UI)** ו**צד שרת (API)**. כשאין משימה ספציפית — לוקחים את השלב הפתוח הראשון לפי הסדר. אחרי כל משימה שהושלמה מעדכנים כאן ☐ → ✅.

**מקרא:** ✅ הושלם · 🔄 בתהליך `[מי, מתי]` · ☐ פתוח

**עבודה במקביל:** התפיסה היא ברמת **שלב שלם** — כל סוכן תופס שלב אחד (מסמן 🔄 עם שמו והתאריך בכותרת השלב, commit + push מיד) ועובד רק על משימות מתוכו. שלב עם 🔄 של סוכן אחר — לא נוגעים בו ולא בקבצים שלו. הפרוטוקול המלא ב-[AGENT_PROMPT.md](AGENT_PROMPT.md).

**רקע:** מסמכי החזון המלאים נמצאים אצל בעלת המוצר; המסמך הזה הוא התרגום המבצעי שלהם. **לכל משימת UI חובה לקרוא קודם את [UI_SPEC.md](UI_SPEC.md)** — אפיון המסכים המלא מהפנקס של בעלת המוצר (מסכים, נוסחים, שדות). שאלה על כוונת המוצר? שואלים את בעלת המוצר, לא מנחשים.

## שלב 3 — פתיחה, הרשמה והגדרת גן וגבייה ☐ (אפיון מלא: UI_SPEC סעיפים 1-7)

**צד לקוח:**
- ☐ מסך פתיחה (Welcome) עם נוסח הפתיחה מהאפיון + "שנתחיל?" / "כבר יש לי חשבון — כניסה" (UI_SPEC ס' 1)
- ☐ מסך כניסה: מייל, סיסמה עם עין, שכחתי סיסמה, כניסה עם Google (UI_SPEC ס' 2) — UI בלבד עד שלב 10 (אבטחה)
- ☐ אשף הרשמה: עיר → שם גן (גם הקלדה חופשית) → מס' ילדים ואנשי צוות → חלוקה לקבוצות (גן: תינוקייה/פעוטות/בוגרים/חובה; בי"ס: א-ו) → עוד ועדים (1-4) (UI_SPEC ס' 3-4)
- ☐ מסך הגדרת גבייה: קטגוריות (הזנה/ועד/חוגים/קלמר אישי) עם סכום לתלמיד + חלוקה ל-1/2/3 תשלומים + סה"כ ויעד כולל (UI_SPEC ס' 5)
- ☐ מסך "הכל מוכן!" עם כרטיס סיכום ההגדרות (UI_SPEC ס' 6)

**צד שרת:** ☐ מודל `Group` (שם, עיר, שנה, מס' ילדים, קבוצות, יעד גבייה) + `CollectionCategory` (קטגוריית גבייה: שם, סכום לתלמיד, מס' תשלומים) + Migration + קשר Student→Group + שכבות מלאות
