# בוט הבדיקות של VaddyGo (E2E)

בוט שנכנס לאתר כמו משתמשת אמיתית ובודק שהכול עובד — שאין קריסות, דפים לבנים או
כפתורים שבורים. רץ בנפרד מבדיקות היחידה (jest); הכלי: [Playwright](https://playwright.dev).

## התקנה (פעם אחת)
```bash
npm install                 # מתקין את Playwright (devDependency)
npx playwright install chromium   # מוריד את הדפדפן שהבוט מפעיל
```

## פרטי חשבון-הבדיקה (סוד — לא בקוד!)
הבדיקות המחוברות משתמשות בחשבון-בדיקה ייעודי. הפרטים מגיעים ממשתני סביבה, לעולם
לא כתובים בקוד:
- **מקומית:** בקובץ `e2e/.env` (מוחרג מ-git). דוגמה:
  ```
  E2E_USER=vaddygo-qa-bot
  E2E_PASSWORD=********
  ```
- **ב-CI:** כ-GitHub Secrets בשמות `E2E_USER` ו-`E2E_PASSWORD`.

## הרצה
```bash
npm run e2e            # בודק את האתר החי
npm run e2e:report     # פותח דוח מפורט (עם צילומי מסך של כישלונות)
```

לבדיקת כתובת אחרת (למשל מקומית):
```bash
E2E_BASE_URL=http://localhost:3000 npm run e2e
```

## מה יש עכשיו
- **בדיקת עשן ציבורית** (`smoke.spec.js`): עמודי כניסה/פתיחה/הרשמה + ניתוב לכתובת
  לא-מוכרת — נטענים, מציגים תוכן (לא דף לבן), ובלי קריסות JavaScript.
- **התחברות** (`auth.setup.js`): מתחבר פעם אחת עם חשבון-הבדיקה ושומר את המצב.
- **בדיקות מחובר** (`loggedin.spec.js`): עובר על העמודים הפנימיים (תלמידים, לוח שנה,
  מתנות, הגדרות, הגדרת גן) — נטענים, לא מעיפים לכניסה, ובלי קריסות (כולל בדיקה
  שמסך השגיאה לא הופיע).

## מה עוד יתווסף (לפי התוכנית)
- **שלב 1 (המשך)** — "צייד הקריסות": לחיצה על כל כפתור/קישור בכל דף.
- **שלב 2** — זרימות שיוצרות נתונים (הוספת תלמיד, ייבוא, תשלום, מתנה) — על חשבון-הבדיקה בלבד.
- **שלב 3** — בדיקת חישובי כספים לעומק.
- **שלב 4** — הרצה אוטומטית בכל שינוי + דוח.

> ⚠️ הזרימות המחוברות יוצרות/מוחקות נתונים — הן ירוצו רק על **חשבון-בדיקה נפרד**,
> לעולם לא על נתונים אמיתיים.
